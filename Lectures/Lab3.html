Task 7: Dictionaries and Queues

 

⍟ Instructions

Create a copy of your project and use the copy for this task.

Instead of having a List<Order>, create a dictionary that stores the Person name and associated Order. That is, an order is stored in the dictionary against an associated name.

A name is then entered into the system to process the Order. For example, if the Dictionary contains a key Alex, and a user enters Alex into a TextBox and clicks a Button, then Alex Order will be processed.

Remember that a Dictionary cannot contain duplicate keys, so you can assume that there will never be any duplicate names. You can code a check to enforce this, if you wish.

Hint

You will need to create a Dictionary<string, Order>

Add a new method to your Order class called ProcessOrder that returns type bool. Its job should call the BuyItem method within the Person class, like so:

public bool ProcessOrder()

{//Person and Item are properties in the Order class

  return Person.BuyItem(Item);

}

Then orders can be processed by calling order.ProcessOrder()

Think about the Queue data structure and how it can be implemented into this system. Instead of instantly processing orders, consider adding these orders to a queue which can then be processed individually by clicking a process order button; this is more in keeping with how a coffee shop would actually process orders.

Research Queue<T> and its associated methods and see how this queued ordering system could be implemented. You do not have to program this unless you want to.

When thinking about this task, consider how a Queue<T> adds and removes items from the queue, and how efficient a Queue<T> compared to a List<T> for particular operations.
